## 1. Industry Terminology Reference

Understanding these terms is critical before interpreting any LMS vendor quote.

### Pricing Model Definitions

| Term | Full Name | Definition | Buyer Impact |
|------|-----------|------------|--------------|
| **Seat** | Named User / Registered User | A specific login credential for a specific person. You pay for every provisioned account whether they log in or not. | Predictable cost, wastes budget on inactive users |
| **MAU** | Monthly Active User | A user counted if they logged in at least once in a given calendar month. Billed monthly, re-assessed each month. | Flexible for seasonal orgs; can spike unpredictably |
| **YAU / AAU** | Yearly Active User / Annual Active User | A user counted if they logged in at least once in the contract year. Assessed at year end or quarterly. | Best for institutions with annual cohorts (universities) |
| **RAU** | Registered Active User | A user who has been registered AND has taken at least one action (viewed a course, submitted an assignment). Stricter than MAU. | Less common; watch for vendor-specific definitions |
| **FTE** | Full-Time Equivalent | Used in higher ed — pricing anchored to enrolled student headcount or employee FTE, not actual logins. | Canvas, D2L commonly use this for universities |
| **Site License** | Unlimited users at a site | Flat fee for all users within a defined institution/org. Common in higher ed for campus-wide licenses. | Simplest for procurement; expensive entry point |
| **Module-Based** | Functionality licensing | Pay for the core LMS + add modules (reporting, AI, compliance, integrations) at additional cost. | Canvas, Cornerstone, D2L heavily modular |

### Pricing Model Evolution (Historical Context)

The LMS pricing model has evolved through three generations:
1. **Pre-2010:** Named seat licenses — pay per provisioned account (Blackboard legacy model)
2. **2010–2018:** Annual Active Users (AAU/YAU) — pay for annual usage, not all registered seats
3. **2018–present:** Monthly Active Users (MAU) + consumption-based models — most aligned with actual usage

**Current industry preference (2025–2026):** YAU for higher education (cohort-based), MAU for corporate training (continuous/rolling learner base). Both are preferable to flat seat licensing from a buyer value perspective.

## 3. Canvas (Instructure)

### Pricing Model
- **Model:** FTE/enrollment-based site license for higher ed; per-user subscription for K–12 and corporate
- **Pricing is entirely custom** — Instructure does not publish list prices
- Contracts are institution-by-institution negotiations

### Price Ranges (Research-Confirmed)

| Segment | Low | Mid | High | Notes |
|---------|-----|-----|------|-------|
| Small institution (<1,000 students) | $5,000–$15,000/yr | $20,000–$35,000/yr | $50,000/yr | Based on $4–$9/student |
| Mid-size university (5,000–15,000 students) | $25,000/yr | $36,000–$70,000/yr | $100,000/yr | Vendr avg: $36K |
| Large university (20,000+ students) | $80,000/yr | $120,000–$200,000/yr | $350,000+/yr | Custom negotiation |
| Per-student benchmark | $4/student | $6–$8/student | $20–$30/student | Lower end for large institutions |
| Add-on: Canvas Studio (video) | +$0.50/user | +$1.08/user | +$2/user | Modular add-on |

**Documented real example:** A public school district paid $6.80/user/year for Canvas LMS + $1.08/user/year for Canvas Studio (public board memo).

**Vendr procurement data:** Average Canvas contract = ~$36,000/year; maximum reported = $140,000/year.

### Pilot Terms
- **Free-for-Teacher account:** Individual educators can create a free account with limited features (no institutional controls, no SSO, no LTI management)
- **Institutional pilots:** No formal public pilot program — requires sales engagement; typically a 30-day sandbox or limited seat proof-of-concept
- **Credit card required for Free-for-Teacher:** No; but institutional pilots require procurement engagement
- **Typical pilot to contract conversion:** 60–90 days discovery + procurement cycle

### Who Pays
- Higher ed: VP Academic / Provost office; IT manages, but budget is academic affairs
- K–12: District IT / Curriculum directors
- Corporate: L&D department or HR Tech budget

### Contract Structure
- **Length:** 3–5 year contracts typical for universities; 1-year for smaller orgs
- **Renewal escalator:** ~3–5% annually (not publicly stated)
- **Implementation fees:** $10,000–$50,000 via Instructure Professional Services or certified partners
- **Setup:** Included in base contract for cloud; migration services billed separately

### Included vs Add-On

| Included in Base | Add-On (Extra Cost) |
|-----------------|---------------------|
| Core LMS (assignments, grades, discussions) | Canvas Studio (video platform) |
| Basic LTI integrations | Canvas Catalog (course marketplace) |
| Mobile apps | Canvas Credentials (digital badges) |
| Analytics (basic) | Canvas Impact (usage analytics) |
| SSO/SAML | Content integrations (external publishers) |

## 5. Blackboard Learn (Anthology)

### Pricing Model
- **Legacy model (Blackboard Learn Classic):** Named seat / perpetual license + annual maintenance (~18–22% of license cost)
- **New SaaS model (Blackboard Learn Ultra):** Per-user bands, annual subscription
- Pricing is entirely custom; no public list price

### Price Ranges (Research-Confirmed)

| Segment | Legacy (On-Prem) | SaaS Ultra | Notes |
|---------|-----------------|-----------|-------|
| Starting price | ~$30,000/yr | ~$9,500/yr | SaaS entry point |
| Mid-market | $50,000–$150,000/yr | $30,000–$80,000/yr | User band pricing |
| Enterprise university | $100,000–$500,000/yr | $80,000–$200,000/yr | Maintenance + support |
| Per-user (UK framework) | £13–£20/user/year | — | Public procurement data |
| Typical range | $30,000–$100,000+/yr | — | educate-me.co research |

**Key distinction:** Blackboard uses **fixed user bands** (e.g., 0–500 users, 501–1,000 users, etc.) rather than per-user pricing. Organizations pay for the band that covers their contracted population — not just active users. This means you pay for who is *eligible* to access the platform, regardless of activity.

### SaaS vs Legacy
- Legacy (Classic) is EOL — Anthology is pushing all customers to Ultra SaaS
- Many large universities remain on Classic due to migration complexity; Anthology charges ongoing support fees
- SaaS Ultra pricing is ~30–50% lower than legacy maintenance + support in many cases

### Pilot Terms
- **30-day free trial** available for Blackboard Learn
- Trial limited in users and features
- Requires sales qualification for enterprise pilots

### Contract Structure
- **Length:** 1–5 years; legacy customers often on multi-year maintenance contracts
- **Migration fees:** $20,000–$100,000 to migrate from Classic to Ultra (Anthology Professional Services)
- **Implementation:** $10,000–$50,000 for new deployments

## 7. TalentLMS

### Pricing Model
- **Model:** Active user-based (users who logged in during the billing month)
- **Published pricing** — one of the few LMS vendors with fully transparent pricing
- Tiers: Free, Core, Grow, Pro, Enterprise

### Published Pricing (2025–2026)

| Plan | Monthly (Billed Monthly) | Monthly (Billed Annual) | User Limit | Key Features |
|------|--------------------------|------------------------|------------|--------------|
| Free | $0 | $0 | 5 users, 10 courses | Test only |
| Core | $139/month | $109/month (~$1,308/yr) | 100 users | Unlimited courses, 1 portal |
| Grow | $299/month | $229/month (~$2,748/yr) | 500 users | Custom reports, 3rd-party integrations |
| Pro | ~$459/month | ~$369/month (~$4,428/yr) | 1,000 users | Advanced features, white-label |
| Enterprise | Custom | Custom | 1,000+ users | Full features, dedicated support |

**Annual savings vs monthly billing:** ~20% discount for annual commitment.

**Per-user effective rates:**
- Core (100 users, annual): ~$13/user/year
- Grow (500 users, annual): ~$5.50/user/year
- Pro (1,000 users, annual): ~$4.40/user/year

### Pilot Terms
- **Free plan available permanently** (up to 5 users, 10 courses) — best-in-class for testing
- No time-limited trial required; free plan lets you evaluate core functionality
- No credit card required for free plan
- Enterprise plan: custom demo + POC period

### Contract Structure
- **Length:** Monthly or annual subscription — most flexible in the market
- **Implementation:** Minimal ($0–$500 for self-service setup); no mandatory onboarding fees
- **No setup fee** for lower tiers — a key differentiator vs enterprise LMS

### Positioned As
Mid-market, SME-friendly, transparent pricing. Popular for corporate training teams of 50–2,000 employees.

### Pricing Model
- **Model:** Custom enterprise contracts; registered user or active user basis depending on negotiation
- No public pricing; entirely sales-led
- Talent suite (LMS + Performance + Recruiting + Skills) sold as modular stack

### Price Ranges (Research-Confirmed)

| Segment | Annual Cost | Notes |
|---------|-------------|-------|
| Average contract (Vendr) | ~$69,000/yr | Procurement database average |

| Customization | $5,000–$20,000 | Integration and workflow customization |
| Total Year-1 TCO (1,000 users) | $80,000–$150,000 | License + implementation + support |

### Government / Defense Pricing
- Cornerstone has a dedicated **Government Edition** (FedRAMP authorized)
- Government contracts typically negotiated through GSA Schedule pricing or direct agency procurement
- Defense/government pricing: **not publicly disclosed**; premium of 20–40% over commercial pricing is industry norm for FedRAMP/ITAR environments
- Cornerstone is one of the few commercial LMS vendors with FedRAMP authorization — significant advantage in the government market

### Pilot Terms
- No formal public pilot or free trial
- Enterprise RFP-driven process: demo → scoped POC (30–60 days) → proposal

### Contract Structure
- **Length:** 3-year contracts standard; some 5-year for government
- **Renewal escalator:** ~5% annually (industry norm)
- Modules priced separately: each talent module (Performance, Succession, Recruiting) adds $10–$40/user/year

## 10. Moodle / MoodleCloud / Moodle Partners

### Pricing Models (Three Distinct Options)

#### A. Moodle LMS (Self-Hosted, Open Source)
- **Software cost:** $0 (GPL license)
- **Actual cost:** Server hosting ($500–$5,000/year) + IT staff time + plugin development
- **True TCO:** $5,000–$50,000/year depending on org size and IT capability
- **No vendor support** included; community forums only

#### B. MoodleCloud (Managed SaaS by Moodle HQ)

| Plan | Annual Price | Users | Storage | Notes |
|------|-------------|-------|---------|-------|
| Starter | ~$160/year | 50 | 1 GB | No plugin support |
| Mini | ~$250/year | 100 | 2 GB | No plugin support |
| Small | ~$410/year | 200 | 5 GB | No plugin support |
| Medium | ~$900/year | 500 | 15 GB | No plugin support |
| Standard | ~$1,970/year | 750 | 50 GB | No plugin support |
| Premium | Custom | 750+ | Custom | Plugin support available |

**Critical limitation:** Standard MoodleCloud plans **do not allow plugins or integrations** — a major restriction for enterprise use.

**Per-user rates:** $2–$3.20/user/year for standard plans.

#### C. Moodle Workplace (Enterprise via Moodle Partners)
- **Moodle Workplace** = enterprise-grade Moodle with multi-tenancy, compliance, programs, certificates
- Sold exclusively through Moodle Certified Premium Partners (e.g., Catalyst IT, Hubken, Lambda Solutions)
- **Pricing: Custom quotes only** — partner markup + hosting + SLA
- Typical range: **$15,000–$80,000/year** depending on users, hosting, and SLA level
- No plugin restrictions — full access to Moodle plugin ecosystem

### Pilot Terms
- MoodleCloud: **28-day free trial** (best-in-class trial length)
- Self-hosted: Unlimited free evaluation (download and run yourself)
- Moodle Workplace: Partner-managed POC; timeline varies by partner

### Who Pays
- Higher ed: IT department (Moodle is an IT-managed platform)
- Corporate: L&D team running on partner-hosted instance
- Government: IT/security teams prefer self-hosted for data sovereignty

## 14. LMS Industry Norms

### Standard Contract Lengths

| Segment | Typical Length | Notes |
|---------|---------------|-------|
| Higher education (enterprise) | 3–5 years | Long procurement cycles, sticky once deployed |
| Corporate enterprise (1,000+ users) | 3 years | Multi-year discounts standard |
| Corporate mid-market (100–1,000 users) | 1–2 years | Shorter, more negotiable |
| SME (< 100 users) | Monthly or annual | Self-service, no lock-in |
| Government/Defense | 3–5 years (often multi-option) | FAR/DFARS compliant contracts |

### Renewal Escalators
- Industry standard: **3–7% annual price increase**
- Most common: 5%
- Multi-year contracts typically lock in escalator rate
- Tip for buyers: Negotiate a cap of 3% in multi-year contracts

### Implementation / Onboarding Fees

| Complexity | Fee Range | What's Included |
|------------|-----------|----------------|
| Self-service (SME) | $0 | Self-onboarding, knowledge base |
| Standard (100–500 users) | $500–$5,000 | Guided setup, SSO config |
| Complex (500–5,000 users) | $5,000–$25,000 | Data migration, SSO, LTI, HRIS |
| Enterprise (5,000+ users) | $25,000–$100,000 | Full migration, custom integrations, training |
| Implementation can equal 20–50% of Year-1 subscription | — | — |

### Pilot Program Standards (2025)

| Pilot Type | Duration | Typical Terms |
|------------|----------|---------------|
| Self-service free trial | 14–30 days | Standard; no credit card for reputable vendors |
| Guided POC (mid-market) | 30 days | Sales-qualified; dedicated CSM support |
| Enterprise pilot | 60–90 days | Limited users (25–100), full features |
| University pilot | 90 days | One department/faculty; full LTI/SSO integration |

**Industry norm 2025:** 30 days is standard; 14 days is considered below market; 90 days is a premium differentiator that significantly accelerates conversion. EduSphere's proposed 90-day pilot is at the top end of the market and is a genuine competitive advantage.

## 15. Compliance Certification Requirements

### What Enterprise Buyers Require (2025 RFP Standard)

| Certification | What It Is | Required By | EduSphere Status |
|--------------|-----------|-------------|-----------------|
| **FERPA** | Family Educational Rights and Privacy Act — US student data privacy | All US higher ed | Implemented (RLS + encryption) |
| **SCORM 2004** | Sharable Content Object Reference Model — content interoperability standard | All LMS buyers | Required (not yet built) |
| **xAPI (Tin Can)** | Experience API — modern learning activity tracking | Enterprise buyers | Required (not yet built) |
| **LTI 1.3** | Learning Tools Interoperability — connect external tools to LMS | Universities | Required (not yet built) |
| **SAML 2.0 SSO** | Federated identity / single sign-on | All enterprise buyers | Implemented (Keycloak) |
| **SCIM** | System for Cross-domain Identity Management — user provisioning | Enterprise/IT buyers | Not built |
| **SOC 2 Type II** | Security audit of cloud service provider controls | Enterprise + government | Not yet (roadmap) |
| **GDPR** | EU data privacy — right to erasure, data portability | EU buyers | Implemented |
| **WCAG 2.1 / 2.2 AA** | Web accessibility — required for public institutions | All institutional buyers | Implemented (Phase 50) |
| **ISO 27001** | Information security management system | Enterprise security teams | Not yet |
| **FedRAMP** | US government cloud security authorization | Federal buyers | Not yet (Cornerstone has this) |
| **ITAR** | International Traffic in Arms Regulations | Defense/aerospace | Not yet |

**Critical insight for EduSphere:** SCORM 2004, xAPI, and LTI 1.3 are **table-stakes** for any university or corporate enterprise sale. Without these, EduSphere cannot pass a standard higher education RFP. These must be prioritized before enterprise sales motions begin.

### Priority Implementation Order
1. **LTI 1.3** — blocks all university deals
2. **SCORM 2004** — blocks all content migration deals
